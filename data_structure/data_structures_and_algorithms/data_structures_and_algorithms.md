## 핵심 요약

- **프로그램 = 자료구조 + 알고리즘**. 자료구조가 결정되면 그에 쓸 수 있는 연산과 알고리즘도 결정된다.
- **ADT(추상 데이터 타입)** 는 데이터와 연산을 "what" 수준에서만 정의하고, "how"는 감춘다 (정보 은닉).
- **알고리즘 성능**은 시간 복잡도와 공간 복잡도로 분석하며, 입력 크기 `n`에 대한 함수로 표현한다.
- **빅오 표기법**은 `n`이 커질 때 가장 영향을 크게 미치는 최고차항만 남긴 점근적 상한이다.

## 1. 자료구조와 알고리즘

### 1.1 자료구조란

- 자료구조란 프로그램에서 처리하는 대상, 자료를 저장하는 구조다.
- 효율적으로 조직화된 자료구조는 프로그램의 효율성을 높인다.
- 더 정확히는, 자료구조는 다음 세 가지의 집합이다:
  - 데이터 값들의 모음 (a collection of data values)
  - 그들 사이의 관계 (the relationships among them)
  - 데이터에 적용할 수 있는 함수나 연산 (the functions or operations)

> **프로그램 = 자료구조 + 알고리즘**
> 예) 최대값 검색 프로그램 = 리스트(자료구조) + 순차검색(알고리즘)

자료구조가 결정되면 그 자료구조에 사용할 수 있는 연산과 알고리즘이 결정된다. 예를 들어 그래프 자료구조에는 다음과 같은 연산과 알고리즘이 따라온다:

- 그래프 구조를 생성하고 정점과 간선을 insertion / deletion
- 그래프의 Traversal 알고리즘: 깊이우선탐색(DFS), 너비우선탐색(BFS)
- 그래프를 이용한 문제 해결 알고리즘: 스패닝 트리, 최소 스패닝 트리, 최단 거리 문제 등

### 1.2 알고리즘이란

컴퓨터로 문제를 해결하려면 방법을 고안하고 그에 따라 단계적 절차를 작성해야 한다. 이 절차가 알고리즘이다. 즉, **알고리즘은 특정 작업을 수행하는 유한한 명령어들의 집합**이다.

**알고리즘의 5가지 조건**

| 조건 | 설명 |
|---|---|
| 입력 | 0개 이상의 입력이 존재해야 한다 |
| 출력 | 1개 이상의 출력이 존재해야 한다 |
| 명백성 | 각 명령어의 의미는 모호하지 않고 명확해야 한다 |
| 유한성 | 한정된 수의 단계 후에는 반드시 종료되어야 한다 |
| 유효성 | 각 명령어는 실행 가능한 연산이어야 한다 |

### 1.3 알고리즘의 기술 방법

알고리즘은 네 가지 방법으로 기술할 수 있다.

**(1) 자연어** — 인간이 읽기 쉽지만, 단어를 정확히 정의하지 않으면 의미가 모호해질 수 있다.

```
ArrayMax(A, n)
1. 배열 A의 첫 번째 요소를 변수 tmp에 복사
2. 배열 A의 다음 요소들을 차례대로 tmp와 비교하여 더 크면 tmp로 복사
3. 배열 A의 모든 요소를 비교했으면 tmp를 반환
```

**(2) 흐름도 (flow chart)** — 직관적이고 이해하기 쉽지만, 복잡한 알고리즘에선 도표가 너무 복잡해진다.

**(3) 유사 코드 (pseudo-code)** — 가장 널리 쓰이는 방법. 자연어보다 구조적이고 프로그래밍 언어보다 덜 구체적이라서, 구현의 세부사항을 감추고 알고리즘의 핵심에 집중할 수 있다.

```
ArrayMax(A, n)
    tmp ← A[0];
    for i ← 1 to n-1 do
        if tmp < A[i] then
            tmp ← A[i];
    return tmp;
```

> `←` 는 대입 연산자임에 유의.

**(4) 프로그래밍 언어** — 가장 정확한 기술이 가능하지만, 구체적인 구현 사항이 알고리즘의 핵심 내용 이해를 방해할 수 있다. 실제 구현 예시는 [`code/array_max.py`](./code/array_max.py) 참고.

## 2. 추상 데이터 타입

### 2.1 데이터 타입

- **데이터**: 처리의 대상이 되는 모든 것
- **데이터 타입**: 데이터의 종류

```
자료형
├── 기초 자료형:   char, int, float, double
├── 파생 자료형:   배열, 포인터
└── 사용자 정의 자료형: 구조체, 공용체, 열거형
```

데이터 타입은 단순히 "값의 집합"이 아니라, **값의 집합 + 그 값에 적용할 수 있는 연산**이다. 예를 들어 `int` 자료형은:

- 데이터: `{-INT_MIN, ..., -2, -1, 0, 1, 2, ..., INT_MAX}`
- 연산: `+`, `-`, `*`, `/`, `%`, `==`, `>`, `<`

### 2.2 추상 데이터 타입 (ADT)

**자료의 추상화**란 사용자에게 중요한 정보는 강조하고 중요하지 않은 구현 세부 사항은 제거하는 것이다. 즉, 자료의 핵심적인 구조나 동작에만 집중한 간략한 기술이며, 이를 **정보 은닉(information hiding)** 이라고 한다.

**추상 데이터 타입(ADT)** 은 데이터 타입을 추상적(수학적)으로 정의한 것이다.

> 데이터나 연산이 **무엇(what)** 인지는 정의되지만, 데이터나 연산을 **어떻게(how)** 컴퓨터에서 구현할지는 정의되지 않는다.

### 2.3 ADT의 정의

ADT는 두 부분으로 구성된다:

- **객체(objects)**: 추상 데이터 타입에 속하는 객체들
- **연산(operations)**: 객체들 사이의 연산. 이 연산이 ADT와 외부를 연결하는 인터페이스 역할을 한다.

**예시: 자연수 ADT (Nat_No)**

```
Nat_No
객체: 0에서 시작하여 INT_MAX까지의 순서화된 정수의 집합
연산:
    zero()       ::= return 0;
    is_zero()    ::= if (x) return FALSE;
                     else return TRUE;
    add(x, y)    ::= if ((x+y) <= INT_MAX) return x+y;
                     else return INT_MAX;
    sub(x, y)    ::= if (x < y) return 0;
                     else return x-y;
    equal(x, y)  ::= if (x == y) return TRUE;
                     else return FALSE;
    successor(x) ::= if ((x+1) <= INT_MAX) return x+1;
```

`add`가 내부적으로 덧셈 회로를 어떻게 쓰든, 메모리에 어떻게 저장하든 사용자는 신경쓰지 않는다. `add(x, y)`가 무엇을 반환하는지만 알면 된다.

## 3. 알고리즘의 성능 분석

### 3.1 성능 분석의 두 가지 방법

**(1) 수행 시간 측정**

- 실제 수행 시간을 측정
- 알고리즘을 실제로 구현해야 함
- 동일한 하드웨어에서 측정해야 비교 가능

C에서는 보통 `clock()` 함수를 사용한다. `clock_t clock(void);` 는 호출된 시점의 시스템 시각을 `CLOCKS_PER_SEC` 단위로 반환한다.

```c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

void main(void) {
    clock_t start, finish;
    double duration;
    start = clock();
    // 수행시간을 측정하고자 하는 코드
    finish = clock();
    duration = (double)(finish - start) / CLOCKS_PER_SEC;
    printf("%f 초입니다.\n", duration);
}
```

**(2) 알고리즘의 복잡도 분석**

- 직접 구현하지 않고도 수행 시간을 분석할 수 있음
- 알고리즘이 수행하는 **연산의 횟수**를 측정
- 연산 횟수는 보통 입력 크기 `n`의 함수로 표현됨
- **시간 복잡도**: 수행 시 필요한 시간(연산 횟수)을 `n`의 함수로 나타낸 것
- **공간 복잡도**: 수행 시 필요한 메모리 크기를 `n`의 함수로 나타낸 것

### 3.2 시간 복잡도 예시

```
ArrayMax(A, n)
    tmp ← A[0];              // 1번의 대입 연산
    for i ← 1 to n-1 do      // 루프 제어 연산은 제외
        if tmp < A[i] then   // n-1번의 비교 연산
            tmp ← A[i];      // n-1번의 대입 연산 (최대)
    return tmp;              // 1번의 반환 연산

총 연산 수 = 2n (최대)
```

### 3.3 복잡도 분석 예시: n을 n번 더하기

동일한 결과를 내는 세 알고리즘을 비교해보자 (for 루프 제어 연산은 제외).

| 알고리즘 A | 알고리즘 B | 알고리즘 C |
|---|---|---|
| `sum ← n*n;` | `for i ← 1 to n do`<br>&nbsp;&nbsp;`sum ← sum + n;` | `for i ← 1 to n do`<br>&nbsp;&nbsp;`for j ← 1 to n do`<br>&nbsp;&nbsp;&nbsp;&nbsp;`sum ← sum + 1;` |

| 연산 종류 | 알고리즘 A | 알고리즘 B | 알고리즘 C |
|---|---|---|---|
| 대입 연산 | 1 | n | n * n |
| 덧셈 연산 | 0 | n | n * n |
| 곱셈 연산 | 1 | 0 | 0 |
| **전체 연산 수** | **2** | **2n** | **2n²** |

`n`이 커질수록 C는 A, B에 비해 훨씬 많은 연산을 수행한다. 그래프로 보면 A는 상수, B는 선형, C는 2차 함수 형태로 증가한다.

### 3.4 빅오 표기법 (Big-O)

자료의 개수가 많아지면 **차수가 가장 큰 항**이 가장 영향을 크게 미치고, 다른 항들은 상대적으로 무시될 수 있다.

예를 들어 `T(n) = n² + n + 1` 일 때, `n = 1,000` 이면:

- 첫째 항 `n² = 1,000,000` → 전체의 약 99%
- 둘째 항 `n = 1,000` → 전체의 약 1%
- 셋째 항 `1` → 거의 0%

따라서 시간 복잡도 함수에서 **최고차항의 차수만** 남긴 것이 빅오 표기법이다. 위 예시는 `O(n²)`.

**정의**

> 두 함수 `f(n)`과 `g(n)`이 주어졌을 때, 모든 `n ≥ n₀`에 대하여 `|f(n)| ≤ c|g(n)|` 을 만족하는 두 상수 `c`와 `n₀`가 존재하면 `f(n) = O(g(n))` 이다.

- 빅오는 함수의 **상한**을 나타낸다.
- 예) `f(n) = 2n² + 3n + 1`, `g(n) = n²` 일 때, `n ≥ 1`에 대해 `c = 6`이면 `2n² + 3n + 1 ≤ 6n²` 이므로 `O(n²)`.
- 예) `f(n) = 2n + 1` 은 `O(n)`.

### 3.5 빅오 표기법의 종류

| 표기 | 이름 | 비고 |
|---|---|---|
| O(1) | 상수형 | 입력 크기와 무관 |
| O(log n) | 로그형 | 이진 탐색 등 |
| O(n) | 선형 | 순차 탐색 등 |
| O(n log n) | 로그선형 | 효율적인 정렬 알고리즘 |
| O(n²) | 2차형 | 이중 루프 |
| O(n³) | 3차형 | 삼중 루프 |
| O(2ⁿ) | 지수형 | 실용적이지 않음 |
| O(n!) | 팩토리얼형 | 실용적이지 않음 |

**데이터 크기에 따른 연산 횟수 비교**

| 시간복잡도 | n=1 | n=2 | n=4 | n=8 | n=16 | n=32 |
|---|---|---|---|---|---|---|
| 1 | 1 | 1 | 1 | 1 | 1 | 1 |
| log n | 0 | 1 | 2 | 3 | 4 | 5 |
| n | 1 | 2 | 4 | 8 | 16 | 32 |
| n log n | 0 | 2 | 8 | 24 | 64 | 160 |
| n² | 1 | 4 | 16 | 64 | 256 | 1,024 |
| n³ | 1 | 8 | 64 | 512 | 4,096 | 32,768 |
| 2ⁿ | 2 | 4 | 16 | 256 | 65,536 | 4,294,967,296 |
| n! | 1 | 2 | 24 | 40,326 | 20,922,789,888,000 | 26313 × 10³³ |

`2ⁿ`과 `n!`은 `n`이 조금만 커져도 다루기 불가능한 수치로 폭발한다.

### 3.6 빅오 이외의 점근 표기법

**빅오메가 (Ω) — 하한**

> 모든 `n ≥ n₀`에 대하여 `|f(n)| ≥ c|g(n)|` 을 만족하는 두 상수 `c`와 `n₀`가 존재하면 `f(n) = Ω(g(n))` 이다.

예) `f(n) = 2n + 1`, `g(n) = n` 일 때, `n ≥ 1`에 대해 `2n + 1 ≥ n` 이므로 `f(n) = Ω(n)`.

**빅세타 (Θ) — 상한과 하한 동시**

> 모든 `n ≥ n₀`에 대하여 `c₁|g(n)| ≤ |f(n)| ≤ c₂|g(n)|` 을 만족하는 세 상수 `c₁`, `c₂`, `n₀`가 존재하면 `f(n) = Θ(g(n))` 이다.

- `f(n) = O(g(n))` 이면서 `f(n) = Ω(g(n))` 이면 `f(n) = Θ(g(n))` 이다.
- 예) `f(n) = 2n + 1`, `g(n) = n` 일 때, `n ≥ 1`에 대해 `n ≤ 2n + 1 ≤ 3n` 이므로 `f(n) = Θ(n)`.

### 3.7 공간 복잡도

공간 복잡도는 알고리즘 수행 시 필요한 메모리의 크기를 `n`의 함수로 나타낸 것이다. 메모리 사용량은 두 부분으로 나뉜다:

- **고정된 공간**: 데이터와 관계없이 필요한 공간 (프로그램 코드, 상수 등)
- **변하는 공간**: 데이터 개수에 따라 변하는 공간

**예시**

```c
#define MAX_ELEMENTS 100
int list[MAX_ELEMENTS];

int sum(int list[], int n) {
    int tmp = 0;
    int i;
    for (i = 0; i < n; i++)
        tmp += list[i];
    return tmp;
}
```

- 프로그램 영역: 고정 사이즈 `P`
- 변수 (`tmp`, `i`, `n`): 3
- 배열 변수: 데이터 크기 `n`
- `S(n) = n + P + 3`

### 3.8 최선 / 평균 / 최악의 경우

알고리즘의 수행 시간은 입력 자료 집합에 따라 달라질 수 있다.

- **최선의 경우 (best case)**: 수행 시간이 가장 빠른 경우 → 의미 없는 경우가 많음
- **평균의 경우 (average case)**: 수행 시간이 평균적인 경우 → 계산이 상당히 어려움
- **최악의 경우 (worst case)**: 수행 시간이 가장 늦은 경우 → **가장 널리 사용됨**. 계산이 쉽고, 응용에 따라 중요한 의미를 가짐 (예: 비행기 관제, 게임, 로보틱스)

**예시: 순차 탐색**

| 경우 | 조건 | 복잡도 |
|---|---|---|
| 최선 | 찾는 값이 맨 앞에 있음 | O(1) |
| 최악 | 찾는 값이 맨 뒤에 있음 | O(n) |
| 평균 | 각 요소가 균일하게 탐색된다고 가정 → `(1+2+...+n)/n = (n+1)/2` | O(n) |

## 4. 자료구조의 구현

### 4.1 C 언어 구현

- 자료구조와 관련된 데이터들을 **구조체**로 정의한다.
- 연산을 호출할 때 이 구조체를 함수의 파라미터로 전달한다.

```c
// 자료구조 스택과 관련된 자료들을 정의
typedef int element;
typedef struct {
    int top;
    element stack[MAX_STACK_SIZE];
} StackType;

// 자료구조 스택과 관련된 연산들을 정의
void push(StackType *s, element item) {
    if (s->top >= (MAX_STACK_SIZE - 1)) {
        stack_full();
        return;
    }
    s->stack[++(s->top)] = item;
}
```

### 4.2 Python 구현

Python에서는 클래스로 자료구조를 정의한다. 데이터와 연산을 하나의 객체에 묶을 수 있어 C의 "구조체 + 함수" 패턴보다 자연스럽다.

구현 전체는 [`code/list_stack.py`](./code/list_stack.py) 참고.

```python
class ListStack_(object):
    def __init__(self):
        self.__items = []

    @property
    def top(self):
        return None if self.is_empty() else self.__items[-1]

    @property
    def size(self):
        return len(self.__items)

    def is_empty(self):
        return len(self.__items) == 0

    def push(self, *data):
        self.__items.extend(data)

    def pop(self):
        return self.__items.pop()
```
