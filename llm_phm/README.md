## 프로젝트 개요

이 프로젝트는 두 가지 축으로 구성됩니다.

**basic_llm_for_domain.ipynb**는 LLM의 기본 활용 패턴을 다룹니다. 멀티턴 대화, 단발 질의, Pipeline API, Few-shot 프롬프팅 기법을 기계공학(재료역학, 진동 신호 분석, 베어링 결함 진단) 도메인에 적용하여, 소형 LLM이 전문 분야에서 어떻게 동작하는지 확인합니다.

**llm_using_rag_with_bearing_data.ipynb**는 KAIST 베어링 진동 데이터를 FFT로 분석하고, RAG(Retrieval-Augmented Generation) 기반으로 유사 사례를 검색한 뒤, LLM이 결함 진단 보고서를 생성하는 엔드투엔드 파이프라인입니다.

두 파일 모두 **GPU 8GB 환경에서 외부 API 없이 로컬 모델**로 진행하였습니다.

## 모델 선정 근거

### 왜 Qwen2.5-3B-Instruct인가

RTX 4060의 8GB VRAM이라는 제약 조건에서, 실용적으로 사용 가능한 최대 크기의 모델을 선택하는 것이 핵심이었습니다.

| 모델 | FP16 VRAM | 8GB 동작 | 비고 |
|------|-----------|----------|------|
| Qwen2.5-1.5B | ~3GB | 여유 | 품질이 용도 대비 부족 |
| **Qwen2.5-3B** | **~6–7GB** | **안정** | **KV 캐시 포함 8GB 내 동작** |
| Qwen2.5-7B | ~17GB | 불가 | 양자화 없이는 실행 불가 |

Qwen2.5-3B-Instruct를 선택한 이유는 다음과 같습니다.

- FP16 추론 시 모델 가중치가 약 6GB로, 8GB VRAM 내에서 안정적으로 동작합니다.
- 한국어 지원으로, 한국어 진단 보고서 생성에 적합합니다.
- Instruction-tuned 모델로 프롬프트 지시 수행 능력이 우수하여, Few-shot이나 구조화된 보고서 생성에 유리합니다.

## basic_llm_for_domain.ipynb - LLM 기본 활용 실습

### 구성

이 파일은 LLM을 활용하는 4가지 패턴을 기계공학 도메인에 적용합니다.

### 1) 멀티턴 대화형 챗봇

```
사용자: 안녕
AI: 안녕하세요! 무엇을 도와드릴까요?

사용자: 기계공학의 기초 역학에 대해서 간략하게 알려줘
AI: 네, 기계공학의 기초 역학에 대해 설명드리겠습니다.

기계공학의 기본 개념 중 하나인 역학은 물질의 움직임과 그 요소들 사이의 관계를 연구하는 분야입니다. 주요 측면으로는 다음과 같은 것들이 있습니다:

1. **유동성**: 유체(가스와 액체)의 움직임과 흐름 특성을 이해하는 것을 포함합니다.
2. **회전력 및 축력**: 회전부품이 발생시키는 힘을 분석합니다.
3. **중력**: 물체가 떨어지는 방식이나 지구를 둘러싼 공이 이동하는 방식 등을 이해하는 것입니다.
4. **운동 법칙**: 에탈-뉴턴 운동 법칙과 같이 물리적 현상을 설명하는 기본적인 운동 법칙들을 배웁니다.

이러한 원리를 기반으로 기계 설계와 작동 동작을 예측하고 개선하는데 활용됩니다. 이러한 원리는 다양한 부품들의 동작을 이해하고 조립하여 전체 기계의 성능을 최적화하는데 중요하게 사용됩니다.

사용자: 운동법칙에 대해서 자세히 알려줘
AI: 물론입니다. 운동 법칙은 기계공학에서 매우 중요한 개념 중 하나이며, 특히 에탈-뉴턴 운동 법칙과 가속도 법칙이 가장 잘 알려진 두 가지 입니다. 

### 1. 에탈-뉴턴 운동 법칙 (Newton's Laws of Motion)
에탈-뉴턴 운동 법칙은 물체가 어떤 상태에서 다른 상태로 변화할 때 필요한 힘과 변형된 속도를 설명합니다. 이 법칙은 세 가지로 나뉩니다:

#### 1.1 첫 번째 운동 법칙 (Law of Inertia)
이 법칙은 "모든 물체는 자연스럽게 그대로 있게 유지하려고 한다"라는 내용을 담고 있습니다. 즉, 무언가가 멈추거나 움직일 때까지는 그 상태를 유지하려 합니다. 이를 통해 물체의 운동 방향이나 속도가 변하지 않도록 만드는 것이 가능해집니다.

#### 1.2 두 번째 운동 법칙 (F = ma)
두 번째 운동 법칙은 "모든 물체에는 그 물체의 질량(m)과 가속도(a) 사이에 일정한 비율이 있다"는 내용을 담고 있습니다. 이 법칙은 힘이 곧 질량과 가속도의 곱이라는 의미를 전달합니다. 이를 수학적으로 표현하면 F = ma로 나타낼 수 있습니다....

```

대화 기록을 누적하며 문맥을 유지합니다. 이전 질문의 "기계공학"을 기억하고 후속 질문인 "운동법칙"과 연결하여 답변합니다. `apply_chat_template`으로 Qwen2.5의 ChatML 형식에 맞게 토큰화하며, VRAM 관리를 위해 대화 기록이 11개(system 1 + 5턴)를 초과하면 오래된 대화를 자동으로 정리합니다.

### 2) 단발 질의 - 재료역학 (응력-변형률 선도)

응력-변형률 선도에서 항복점, 극한강도, 파단점 각각의 물리적 의미와 기계 설계 시 안전계수와의 관계를 하나의 질문으로 설명하도록 요청합니다. `do_sample=False`(greedy decoding)로 재현 가능한 결정론적 출력을 보장하여, 교과서 수준의 일관된 설명을 기대할 수 있습니다.

### 3) Pipeline 기반 멀티턴 챗봇 - FFT 이상치 탐지

회전 기계의 진동 시계열 신호에서 FFT를 이용한 이상치 탐지 방법론을 멀티턴으로 심화 탐구합니다.

```
질문 1: 회전 기계의 진동 시계열 신호에서 FFT를 이용하여 이상치를 탐지하는 기본 원리를 설명해줘.
        정상 상태와 이상 상태의 주파수 스펙트럼이 어떻게 다른지,
        그리고 왜 시간 영역보다 주파수 영역 분석이 결함 탐지에 유리한지도 포함해줘.

질문 2: (이전 답변을 기반으로)
        그러면 FFT 스펙트럼에서 구체적으로 어떤 패턴이 나타날 때
        이상치로 판단해? 예를 들어 특정 주파수에서 피크가 얼마나 커야
        이상으로 볼 수 있는지, 하모닉이나 사이드밴드 같은 패턴은
        각각 어떤 종류의 결함을 의미하는지 설명해줘.
```

첫 번째 질문에서 FFT 기반 이상치 탐지의 전체적 원리를 묻고, 두 번째 질문에서 이전 답변의 맥락을 이어 구체적 판별 기준(하모닉 피크, 사이드밴드 패턴)으로 파고듭니다. 이 내용은 llm_using_rag_with_bearing_data.ipynb의 `calculate_bearing_power` 함수가 실제로 수행하는 작업과 직결되므로, 이론적 배경과 구현의 연결을 보여줍니다.

### 4) Few-shot 프롬프팅 - 베어링 결함 위치 분류

FFT 스펙트럼 분석 결과(BPFI/BPFO/BSF 하모닉 패턴과 RMS 수치)를 바탕으로 베어링 결함 위치를 분류합니다. 동일한 분류 문제를 Zero-shot과 Few-shot으로 풀어 결과를 비교합니다.

**Zero-shot** - 예시 없이 바로 분류를 요청합니다.

**Few-shot** - 2개의 예시(외륜 결함, 정상)를 대화 기록으로 제공한 뒤 분류를 요청합니다.

```
예시 1: "BPFO 하모닉에서 에너지 집중, RMS=0.051" → 외륜 결함 (O)
        이유: 외륜에 국부적 결함(점식, 박리) 존재, ISO 10816 기준 '주의'

예시 2: "전 주파수 대역 에너지 균일, 피크 없음, RMS=0.006" → 정상 (N)
        이유: 결함 주파수에서 유의미한 피크 없음, 노이즈 플로어 수준

분류 문제:
  1) BPFI 2~5차 하모닉에서 뚜렷한 피크, RMS=0.045  → ?
  2) 전 대역 에너지 균일하게 낮음, RMS=0.008       → ?
  3) BPFO 1차 및 3차 하모닉에서 높은 에너지, RMS=0.062 → ?
```

Few-shot 예시에 ISO 10816 기준, 피팅/박리 같은 구체적 결함 유형, 노이즈 플로어 개념을 포함하여 모델이 단순 패턴 매칭이 아닌 진동 분석 지식 기반의 추론을 수행하도록 유도합니다. llm_using_rag_with_bearing_data.ipynb에서 수행하는 베어링 상태 판단을, 여기서는 LLM의 Few-shot 기법으로 시도하는 것입니다.

### 핵심 기술 요소

| 기술 | 적용 방식 |
|------|-----------|

### STEP 1: 베어링 결함 주파수 계산

`BearingFaultCalculator` 클래스는 베어링의 기하학적 형상으로부터 4가지 결함 특성 주파수를 계산합니다.

볼 베어링은 내륜(Inner Race), 외륜(Outer Race), 볼(Rolling Element), 케이지(Cage)로 구성됩니다. 각 부품에 결함이 발생하면 회전 시 특정 주파수에서 반복적인 충격이 발생하며, 이 주파수는 베어링의 기하 형상과 회전 속도에 의해 결정됩니다.

```
geo_factor = (Bd / Pd) × cos(접촉각)

BPFO = (n/2) × fr × (1 - geo_factor)      <- 외륜 결함
BPFI = (n/2) × fr × (1 + geo_factor)      <- 내륜 결함
BSF  = (Pd/2Bd) × fr × (1 - geo_factor^2) <- 볼 결함
FTF  = (1/2) × fr × (1 - geo_factor)      <- 케이지 결함
```
NSK 6205 깊은 홈 볼 베어링(Deep Groove Ball Bearing) 기준으로, 접촉각은 0도, 볼 직경 약 7.9375mm, 볼 개수 9개입니다. 1780 RPM에서의 계산 결과 예시로 BPFI는 약 160Hz, BPFO는 약 107Hz 수준입니다.

### STEP 2: FFT 기반 진동 파워 분석

`calculate_bearing_power` 함수는 시간 영역 진동 신호를 주파수 영역으로 변환하여 결함 주파수의 에너지를 정량화합니다.

처리 과정은 다음과 같습니다.

1. **DC 제거** - 신호에서 평균값을 빼 정적 오프셋 제거
2. **FFT 변환** - 128,000포인트(5초 @ 25,600Hz) 신호를 주파수 영역으로 변환
3. **파워 스펙트럼 계산** - 단측 스펙트럼에 Parseval 정규화 적용
4. **하모닉 파워 추출** - 결함 주파수의 1~19차 하모닉 중 1000~3000Hz 범위에 있는 성분의 ±5Hz 영역 평균 파워를 계산

결함이 있는 베어링은 해당 결함 주파수의 하모닉에서 에너지가 집중되므로, 이 평균 파워 값이 높을수록 결함 가능성이 큽니다.

### STEP 3: RAG - FAISS 하이브리드 검색

과거 진단 사례 데이터(JSON)에서 현재 측정 데이터와 유사한 사례를 2단계 하이브리드 검색으로 찾습니다.

**인프라 구성:**
- `paraphrase-multilingual-MiniLM-L12-v2` - 한국어 지원 다국어 임베딩 모델 (CPU 실행)
- `FAISS IndexFlatIP` - 코사인 유사도 기반 벡터 검색 인덱스

**FAISS + 수치 오차:**

```
현재 측정 데이터
      │
      ▼
┌─────────────────────────────────────────┐
│ 1단계: FAISS 의미 검색 (넓은 그물)       │
│                                         │
│  측정 데이터를 자연어 텍스트로 변환:      │
│  "베어링 진동 분석 결과: 부하 4.0Nm,     │
│   RMS 0.0342, BPFI 파워 1.23e-05, ..."  │
│         │                               │
│         ▼                               │
│  Sentence Transformer로 임베딩           │
│         │                               │
│         ▼                               │
│  FAISS index.search() → 상위 20개 후보   │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│ 2단계: 수치 오차 리랭킹 (정밀 필터)      │
│                                         │
│  20개 후보 내에서:                       │
│  - BPFI/BPFO 절대 오차 → 수치 유사도    │
│  - FAISS 코사인 점수 → 의미 유사도       │
│  - 가중 결합 (의미 0.4 + 수치 0.6)      │
│         │                               │
│         ▼                               │
│  최종 상위 2개 사례 선정                  │
└─────────────────────────────────────────┘
```

1단계에서 FAISS가 "RMS가 높고 BPFI에서 에너지가 집중된" 맥락을 가진 과거 보고서들을 의미적으로 추려내고, 2단계에서 BPFI/BPFO 수치가 실제로 가까운 사례를 최종 선정합니다. 의미 검색만 쓰면 수치 정밀도가 떨어지고, 수치 비교만 쓰면 맥락이 무시되므로, 두 방식의 장점을 결합한 구조입니다.

### STEP 4: LLM 보고서 생성

RAG로 검색된 유사 사례와 현재 측정 데이터를 프롬프트에 구성하여, Qwen2.5-3B-Instruct가 진단 보고서를 생성합니다.

**프롬프트 구조:**

```
[System] 베어링 진동 분석 전문가 역할 지정

[User]
  ├── 작성 지침 (판단 기준, 출력 형식 등)
  ├── 참고 사례 (RAG 검색 결과)
  │   ├── 사례 1: 베어링_상태, 정비 액션 아이템
  │   └── 사례 2: 베어링_상태, 정비 액션 아이템
  └── 입력 데이터 (RMS, BPFI 파워, BPFO 파워)
```

**생성 파라미터:**

보고서 품질을 위해 `temperature=0.1` `repetition_penalty=1.2`를 사용합니다. `return_full_text=False`로 생성된 보고서만 추출합니다.

### 출력 예시

```
[측정 데이터]
  부하: 4.0 Nm
  RPM:  1780
  RMS:  0.0342
  BPFI 파워: 1.234567e-05
  BPFO 파워: 8.765432e-06

[RAG 기반 AI 진단 보고서]
- 베어링_상태: BPFI 결함 (내륜 결함 초기 단계)
- 위험도 평가: 주의
- RMS 수치가 정상 범위 상한에 근접하며, BPFI 평균 파워가
  정상 대비 유의미하게 높음
- 정비 액션 아이템:
  1) 진동 모니터링 주기를 주 1회에서 일 1회로 단축함
  2) 윤활 상태를 점검하고 필요 시 그리스를 재도포함
  3) 30일 이내 베어링 교체를 계획에 포함할 것을 권고함
```

## 핵심 기술 스택

| 영역 | 기술 | 용도 |
|------|------|------|

## 프로젝트 의의

**도메인 특화 로컬 LLM의 가능성 검증** - 8GB GPU라는 일반 소비자급 하드웨어에서도 전문 분야에 LLM을 실용적으로 적용할 수 있음을 보여줍니다. 외부 API 의존 없이 완전 로컬 추론이 가능하여 보안이 중요한 산업 현장에 적합합니다. 하지만 그만큼 제약사항(출력제한, 응답 품질, 문맥 등)이 발생함을 확인하였습니다.

**PHM(Prognostics and Health Management)과 AI의 결합** - 전통적인 신호 처리(FFT, 결함 주파수 분석)와 현대적 AI 기술(RAG, LLM)을 결합한 하이브리드 접근입니다. 수치 분석으로 정량적 판단을, LLM으로 사람이 읽을 수 있는 보고서를 생성합니다.

**Few-shot Learning의 공학 적용** - basic_llm_for_domain.ipynb의 Few-shot 분류와 llm_using_rag_with_bearing_data.ipynb의 RAG 기반 진단이 동일한 베어링 도메인을 다루어, 접근 방식의 차이와 상호 보완성을 보여줍니다.
