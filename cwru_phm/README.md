# CWRU 베어링 데이터 기반 PHM 신호처리

> 진동 신호처리 기초부터 고장 진단, 수명 예측까지
> CWRU 실제 데이터로 각 기법이 왜 필요한지, 어디서 한계가 생기는지를 확인하는 개인 학습용

---

## 참고 사항

이 프로젝트의 신호처리 기법은 25년 한국PHM학회의 **PHM 리더양성을 위한 전문기술 강좌**에서 진행된 예제 코드를 기반으로 재구성하였습니다.

- 베어링 결함 주파수 계산 공식 및 기하학적 파라미터는 [CWRU 공식 페이지](https://engineering.case.edu/bearingdatacenter/bearing-information)의 스펙을 교차 검증하여 사용하였습니다.
- 데이터는 [CWRU Bearing Data Center](https://engineering.case.edu/bearingdatacenter)에서(혹은 Kaggle) 제공하는 .mat 파일을 사용합니다.

## 데이터: CWRU Bearing Dataset

### 어떻게 수집된 데이터인가

CWRU 데이터는 2HP Reliance Electric 유도 모터에 SKF 6205-2RS JEM 볼 베어링을 장착하고, 방전 가공(EDM)으로 인공 결함을 만든 뒤 진동을 측정한 것입니다.

![수집환경](https://engineering.case.edu/sites/default/files/styles/page_image/public/CWRU_teststand.jpeg?itok=rRXOZhHU)

(출처 [Apparatus &amp; Procedures | Case School of Engineering](https://engineering.case.edu/bearingdatacenter/apparatus-and-procedures))

가속도계는 모터 하우징 12시 방향(꼭대기)에 자석 베이스로 부착하고, 구동단(DE)과 팬단(FE) 양쪽에서 12,000Hz로 샘플링합니다. 외륜 결함은 부하 영역 대비 위치(6시/3시/12시)에 따라 진동 응답이 달라지는데, 본 데이터는 모두 **6시 방향(부하 영역 중심, 최대 충격)**입니다.

모터 부하는 1HP이고, 유도 모터의 슬립에 의해 회전 속도는 동기 속도(1,800 RPM)보다 약간 낮은 **~1,772 RPM**입니다. 파일마다 RPM이 1771~1774로 약간 다른 것은 정상적인 편차이며, 파이프라인에서는 대표값 1772를 사용합니다.

### 사용한 파일 (10개)


| 파일명                  | 결함      | 크기   | RPM  | 특이사항                  |
| ------------------------- | ----------- | -------- | ------ | --------------------------- |

### 결함 주파수 계산과 스펙 검증

베어링의 스펙(볼 9개, Bd=7.94mm, Pd=39.04mm, 접촉각 0도)에서 결함 주파수를 계산하고, CWRU 공식 페이지에 공개된 배수값과 대조 검증하였습니다.


| 주파수    | 약어   | CWRU 공식 배수 | 계산값    | 오차    |
| ----------- | -------- | ---------------- | ----------- | --------- |
| 외륜 결함 | BPFO   | 3.5848×       | 105.88 Hz | 0.01 Hz |
| 내륜 결함 | BPFI   | 5.4152×       | 159.92 Hz | 0.01 Hz |
| 케이지    | FTF    | 0.39828×      | 11.76 Hz  | 0.00 Hz |
| 볼 충격   | 2×BSF | 4.7135×       | 139.24 Hz | 0.03 Hz |

검증 과정에서 발견한 점: CWRU의 "Rolling Element" 배수(4.7135)는 볼 자전 주파수(BSF)가 아니라 **2\*BSF**입니다. 볼이 1회전 시 내·외륜과 각각 1회씩 접촉하여 2회 충격이 발생하기 때문에, 실제 결함 충격 주파수는 2*BSF입니다.

## 분석 전 예상

데이터를 돌리기 전에, 수집 환경과 기초 도메인으로부터 각 단계에서 어떤 결과가 나올지 미리 예상해 보았습니다.


| 단계           | 예상                                                           | 근거                                                                               |
| ---------------- | ---------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| FFT            | 결함 주파수 자체보다 구조적 공진(수백 Hz~수 kHz)에 에너지 집중 | 충격이 고유 주파수를 여기시키는 것이지, 결함 주파수 자체가 강하게 나오는 것은 아님 |
| FFT            | OR 진폭 최대, Ball 진폭 최소                                   | OR은 6시 방향 최대 부하에서 충격, Ball은 접촉 불규칙으로 에너지 분산               |
| Envelope       | OR→BPFO 가장 선명, Ball→2×BSF 가장 약함                     | 외륜 고정 → 일정한 충격 진폭, 볼 자전 → 불규칙 접촉                              |
| Classification | Inner↔Ball 혼동 최대                                          | 둘 다 회전 부품에서 발생 → 유사한 진폭 변조 패턴                                  |
| Prognosis      | 비단조 열화로 모델 피팅 실패                                   | CWRU는 snapshot 데이터이고, OR007 RMS가 OR021보다 클 수 있음                       |

## 단계별 결과와 확인한 것

### Step 1. FFT

시간 영역 신호를 주파수 영역으로 변환하여 에너지 분포를 확인하였습니다.

**확인한 것**: 예상대로 결함 주파수(BPFO 106Hz, BPFI 160Hz)는 FFT에서 직접 보이지 않고, 700~900Hz의 구조적 공진에 에너지가 집중됩니다. OR007의 RMS가 정상의 16배(1.061 vs 0.066)로 가장 컸고, Ball007이 2.1배(0.141)로 가장 작았습니다.

**여기서 알게 된 것**: Raw FFT만으로는 "결함이 있다"는 것은 알 수 있지만, "어디에 결함이 있는지"(내륜인지 외륜인지 볼인지)는 특정할 수 없습니다. 이것이 Envelope Analysis가 필요한 이유입니다.

### Step 2. Filtering

Butterworth LPF(500Hz)로 고주파 노이즈를 제거하고, BPF(2~5kHz)로 베어링 공진 대역만 추출하였습니다.

**확인한 것**: 정상 신호는 LPF 전후 거의 차이가 없고(대부분의 에너지가 이미 저주파), 결함 신호는 고주파 성분 제거로 진폭이 크게 감소합니다. 이 차이 자체가 고주파 충격 에너지의 존재를 간접적으로 보여줍니다.

### Step 3. Envelope Analysis

이 스터디에서 가장 핵심적인 부분입니다. BPF(2~5kHz) → 힐버트 변환 → 포락선 추출 → FFT 순서로 결함 주파수를 검출합니다.

**확인한 것**:


| 결함    | 이론 주파수     | 검출 피크 | SNR    | 예상과 일치? |
| --------- | ----------------- | ----------- | -------- | -------------- |
| OR007   | BPFO 105.88Hz   | 106.2Hz   | 13.9배 | ✅ 가장 선명 |
| IR007   | BPFI 159.92Hz   | 159.4Hz   | 16.7배 | ✅           |
| Ball007 | 2×BSF 139.24Hz | 136.6Hz   | 4.9배  | ✅ 가장 약함 |

Raw FFT에서 전혀 보이지 않던 결함 주파수가, 포락선 스펙트럼에서는 노이즈 플로어 대비 14\~17배의 피크로 명확하게 나타납니다. 이론값과 검출값의 차이(0.3~2.6Hz)는 볼의 슬립에 의한 것으로 판단되며, 일반적으로 발생하는 현실적인 현상으로 알려져있습니다.

**추가 발견**: 외륜 결함 크기별(OR007→OR014→OR021) 비교에서, OR014의 BPFO 피크(SNR 3.0배)가 OR007(13.9배)보다 오히려 약했습니다. 결함이 커졌는데 신호가 약해지는 비단조적 경향인데, 이는 결함 크기 외에 베어링 재설치 상태, EDM 가공 형태가 영향을 미치기 때문입니다.

### Step 4. STFT

스펙트로그램으로 시간-주파수 동시 분석을 수행하였습니다.

**확인한 것**: CWRU는 정속 운전이므로 수평 밴드가 기본이지만, OR007에서는 주기적 충격에 의한 **세로 줄무늬**가 뚜렷하게 관찰됩니다. 이 줄무늬가 3kHz 이상까지 뻗어 있어, 충격 에너지가 매우 넓은 대역에 퍼지는 것을 확인합니다. Ball007의 스펙트로그램은 Normal과 거의 구분이 안 됩니다.

**여기서 알게 된 것**: 세로 줄무늬가 광대역에 걸쳐 있다는 것은, 포락선 분석에서 BPF 대역을 어디로 잡든(2\~3kHz든 3~4kHz든) 결함 검출이 가능하다는 물리적 근거가 됩니다.

### Step 5. Classification

시간·주파수·포락선 3개 도메인에서 14개 특징을 추출하여 4클래스(Normal/Inner/Outer/Ball) 분류를 수행하였습니다.

**확인한 것**: Random Forest **95.24%** (5-Fold CV 평균 **95.71 ± 2.67%**). 오분류 2건 모두 Inner↔Ball 간이었고, Normal과 Outer는 100% 정확합니다. 예상대로입니다.

Feature Importance 상위: P2P(15.5%), FFT_200_500(13.4%), RMS(13.1%), Env_200_500(12.3%)로, 시간/주파수/포락선 세 도메인이 고루 기여합니다. 포락선의 결함 주파수별 특징(Env_BPFO, Env_BPFI 등)은 개별적으로는 중요도가 낮지만, Env_0_200과 Env_200_500이 이들을 포괄하여 대역 에너지가 더 효과적인 특징임을 확인하였습니다.

### Step 6. Prognosis

Normal → OR007 → OR014 → OR021의 RMS를 시간순으로 연결하여 의사 열화 궤적을 구성하고, 지수 모델을 피팅하였습니다.

**발생한 문제**: 열화 궤적의 RMS가 0.066 → **1.060** → 0.146 → 0.630으로 급등-급락-재상승 패턴을 보여, 그래프의 y축이 1e15 스케일로 발산하면서 사실상 직선으로 보입니다. RUL 불확실성도 0으로 수렴합니다.

**원인**: **CWRU 데이터의 구조적 한계**입니다.

1. OR007/OR014/OR021은 동일 베어링의 열화가 아니라 **서로 다른 베어링의 독립 실험**
2. 결함 크기와 진동 에너지가 **단조 관계가 아님** (OR007 RMS 1.060 > OR021 0.630)
3. 지수 모델은 **단조 증가를 전제**로 하므로, 올라갔다 내려가는 데이터에서는 계수가 발산

**여기서 알게 된 것**: 문제는 방법론이 아니라 **데이터 특성과 모델 가정의 불일치**입니다. 실무에서도 Prognosis가 실패하는 가장 흔한 원인이 이것이고, 사전에 데이터가 모델 가정(단조 증가)을 만족하는지 확인하는 과정이 필수적입니다. 실제 RUL 예측에는 동일 베어링의 run-to-failure 데이터가 필요합니다.

## 학습을 통해 배운 것

**포락선 분석이 왜 필요한지 체감하였습니다.** Raw FFT로는 안 보이던 결함 주파수가 포락선 스펙트럼에서 명확하게 나타나는 것을 직접 확인하였습니다.

**도메인 지식으로 결과를 예상하는 습관의 중요성을 알게 되었습니다.** 데이터를 돌리기 전에 "OR이 가장 클 것이다", "Inner-Ball이 헷갈릴 것이다" 같은 예상을 세워두니, 결과가 나왔을 때 맞는지 틀리는지를 바로 판단할 수 있었고, 예상과 다를 때 원인을 추적하는 방향이 잡혔습니다.

**모델 가정과 데이터 특성의 일치 여부를 사전에 확인해야 한다는 교훈을 얻었습니다.** Step 6에서 그래프가 직선으로 나왔을 때, 처음에는 코드 버그를 의심했지만 실제로는 CWRU snapshot 데이터의 비단조적 특성과 지수 모델의 단조 증가 가정이 불일치한 것이었습니다.
