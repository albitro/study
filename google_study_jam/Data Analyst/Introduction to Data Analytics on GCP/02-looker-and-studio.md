## 1. Looker 소개

### 1.1. Looker란?

- 회사가 수집한 데이터에 액세스하고 검토하는 **강력한 분석 도구**
- **실시간**으로 데이터 질문에 답을 얻을 수 있음
- 데이터 기반 **의사결정** 지원

### 1.2. Looker의 데이터 분석 프로세스

- **답을 찾고 싶은 질문 파악** (예: 지난주 매출, 사용자 인구통계, 배송 상태)
- **필요한 데이터 식별** (예: 주문, 배송, 유통 센터 데이터)
- **Looker에서 분석(Explore)**: 측정기준 + 측정 결합
- **결과 해석 및 시각화** → 인사이트 도출 → 의사결정

## 2. Looker로 데이터 분석 및 차트 작성

### 2.1. Explore (보고서 작성 도구)

- 데이터를 사용해 질문할 수 있는 **포털**
- **LookML**(독점 템플릿 언어)로 Looker 개발자가 선별
- 액세스 권한이 부여된 Explore만 표시됨

### 2.2. 핵심 필드: 측정기준 vs 측정


| 구분                     | 설명                                     | 예시                         |
| -------------------------- | ------------------------------------------ | ------------------------------ |
| **측정기준 (Dimension)** | 데이터의 속성/특성. DB 테이블의 각**열** | 주문 날짜, 판매가, 배송 상태 |
| **측정 (Measure)**       | 여러 행에서 수행되는**계산/집계**        | 개수(count), 합계(sum)       |

> 💡 **활용 팁**: 데이터 질문의 답을 얻기 위해 측정기준과 측정을 **결합**해서 사용

### 2.3. 데이터 탐색 기법

#### 2.3.1. 필터(Filter)

- 특정 기준에 따라 결과 범위를 좁힘
- 측정기준과 측정 모두 필터링 가능
- **DB의 데이터를 삭제하지 않음** → Looker 화면 표시에만 적용
- 예시: 미국 전체 도시 → "캘리포니아주" 필터 적용

#### 2.3.2. 피벗(Pivot)

- 선택한 측정기준을 **여러 개의 열로 변환** → 데이터 행렬 생성
- 측정기준의 고유값이 각각 **열 헤더**가 됨
- 그룹별 비교 및 격차 파악에 유용
- 기본 데이터에 영향 없음

### 2.4. 시각화 및 저장

#### 2.4.1. 시각화 유형

- 열 차트, 막대 차트 등 다양한 옵션
- 값 라벨 표시 등 커스터마이징 가능

#### 2.4.2. Look (룩)

- **저장된 단일 보고서 또는 시각화**
- 개별 데이터 질문에 대한 답 제공
- 대시보드에 추가 가능

#### 2.4.3. 대시보드(Dashboard)

- **여러 보고서를 한 페이지**에 표시
- 여러 데이터 질문에 대한 답 제공
- 데이터 발견 항목을 저장하는 공간으로 활용

## 3. Looker Studio로 데이터 시각화

### 3.1. Looker vs Looker Studio 차이점

- **Looker**: 강력한 분석 + 보고서 분석 기능
- **Looker Studio**: 분석 기능은 적지만 **시각화에 집중**한 간편한 인터페이스

### 3.2. Looker Studio의 장점

- **드래그 앤 드롭 인터페이스**
- 다양한 보고서/대시보드 (차트, 그래프, 테이블)
- 필터 추가, 데이터 상세 분석 등 전문가 수준의 보고서
- **실시간 협업**
- 액세스 사용자 제어 보안 기능

### 3.3. Looker 커넥터를 통한 통합

- Google Cloud 호스팅 Looker 인스턴스에 연결
- 사용 절차:
  - Looker 인스턴스에서 통합 사용 설정
  - Looker Studio에서 새 보고서 생성
  - 데이터 소스로 Looker 커넥터 선택
  - Looker 인스턴스 URL 입력 및 Explore 선택

### 3.4. 시각화 선택 시 고려사항

- 작업 중인 **데이터 유형**은 무엇인가?
- 어떤 **인사이트**를 전달하려고 하는가?
- 보고서의 **대상**은 누구인가?

### 3.5. 시각화 옵션별 용도


| 시각화 유형 | 용도                        |
| ------------- | ----------------------------- |
| 막대 그래프 | 다양한 카테고리 데이터 비교 |
| 선 차트     | 시간에 따른 데이터 변화     |
| 영역 차트   | 선 차트 + 아래 영역 채움    |
| 원형 차트   | 카테고리 상대적 비율        |
| 도넛 차트   | 원형 차트 + 중앙 구멍       |
| 분산형 차트 | 두 변수 간 관계 표시        |
| 히트맵      | 데이터 분포 표시            |
| 지도        | 지리적 위치 데이터          |

### 3.6. Looker Studio 차트 작성 흐름

- 툴바에서 **'차트 추가'** 클릭
- 차트 유형 선택
- 측정기준 추가 (예: 기간, 분류)
- 필터 적용 (예: 과거 1년)
- 스타일 탭에서 라벨/색상/글꼴 커스터마이징

## 4. 보고서 및 시각화 공유

### 4.1. Looker에서 데이터 공유

#### 4.1.1. 공유 방식

- **Look으로 저장**: 단일 보고서로 저장
- **대시보드에 저장**: 여러 보고서 모음
- **다운로드**: 컴퓨터로 추출 (CSV, ZIP, PDF, PNG)
- **보내기(Send)**: 이메일 등 다른 곳으로 전송
- **일정 예약(Schedule)**: 매일/매주 정기 전송
- **공유(Share)**: URL 복사 후 권한 있는 사용자에게 전달

### 4.2. Looker Studio에서 데이터 공유

- **Google Drive 기반**으로 구축
- 문서/스프레드시트/슬라이드처럼 공유 가능
- **편집 권한** 또는 **보기 권한** 부여
- 공유 위치: Looker Studio 홈페이지 또는 보고서 자체
- 추가 기능: 전송 일정 예약, 보고서 링크 생성, 다운로드

## 5. Looker vs Looker Studio 비교

### 5.1. Looker

#### 5.1.1. 장점

- 커뮤니케이션·협업 개선
- 빠른 의사결정 + 효율성 향상
- **중앙 집중식 저장소** → 데이터 정확성·일관성 보장
- 확장성 우수
- 다른 데이터 소스/애플리케이션과 통합 가능

#### 5.1.2. 단점

- 클라우드 저장 → 보안 침해 취약 가능 (단, 암호화·액세스 제어 제공)
- **유료 플랫폼** (비용 부담)
- 상대적으로 복잡 → 학습 시간 필요

### 5.2. Looker Studio

#### 5.2.1. 장점

- **사용 편의성** (사용자 친화적)
- 유연한 데이터 공유 환경
- 권한 설정 가능
- 보안 기능 (암호화, 액세스 제어)

#### 5.2.2. 단점

- Looker 대비 **성능이 떨어짐**
- 고급 데이터 모델링 / 머신러닝 미지원
- 통합 가능한 데이터 소스가 적음

### 5.3. 선택 기준


| 조직 유형                                    | 추천 플랫폼       |
| ---------------------------------------------- | ------------------- |
| 강력하고 확장 가능한 데이터 공유 플랫폼 필요 | **Looker**        |
| 사용자 친화적인 플랫폼 필요                  | **Looker Studio** |

---

## 6. 핵심 정리

- 6.1. **Looker**는 측정기준 + 측정 결합으로 데이터 질문에 답을 찾는 **분석 중심 플랫폼**이다.
- 6.2. **필터**는 화면 표시에만 영향을 주고, **피벗**은 데이터 행렬을 만들어 비교를 쉽게 한다.
- 6.3. **Look**은 단일 보고서, **Dashboard**는 여러 Look의 모음이다.
- 6.4. **Looker Studio**는 시각화·공유에 특화된 사용자 친화적 도구이며, **Google Drive 기반**으로 협업이 쉽다.
- 6.5. 조직 규모와 요구 사항에 따라 **Looker(고급/확장성)** 또는 **Looker Studio(간편함)**를 선택한다.
